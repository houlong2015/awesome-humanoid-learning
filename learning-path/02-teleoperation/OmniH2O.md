# OmniH2O: Universal and Dexterous Human-to-Humanoid Whole-Body Teleoperation and Learning

> 论文：Tairan He†, Zhengyi Luo†, Xialin He†, Wenli Xiao, Chong Zhang, Weinan Zhang, Kris Kitani, Changliu Liu, Guanya Shi  
> 机构：CMU / 上海交大  
> 主页：[https://omni.human2humanoid.com](https://omni.human2humanoid.com)  
> arXiv：[2406.08858](https://arxiv.org/abs/2406.08858)（2024-06）  
> 标签：`Teleoperation` · 全身遥操作 · 通用接口 · 模仿学习数据集

---

## 一、总结

OmniH2O（Omni Human-to-Humanoid）是一套**学习式全身人形遥操作与自主系统**。以**运动学姿态（kinematic pose）为通用控制接口**，同一套全身运动模仿策略可对接：

- VR 头显实时遥操作  
- RGB 相机  
- 语言指令（经 MDM 等文生动作模型）  
- 前沿多模态模型（如 GPT-4o）  
- 由遥操作数据训出的模仿策略  

在 Unitree H1 + 灵巧手上完成打球、浇花、写字、蹲捡、拳击、搬篮等真机任务；并开源首个全身 loco-manipulation 数据集 **OmniH2O-6**（六项日常任务），示范从遥操作数据学自主技能。

相对前作 H2O：不再依赖测试时 MoCap 全局线速度；用教师-学生蒸馏 + 历史观测，支持稀疏三点（头+双手）输入下的高精度灵巧全身控制。

---

## 二、问题背景

人形作为「通用具身」需要大规模真机演示；但全身控制难，现有工作常只做下肢，或上/下肢解耦。要同时支持**稳定灵巧操作 + 鲁棒移动**，必须端到端协调上下肢。昂贵 MoCap/外骨骼也阻碍规模化采集。

需求归纳：

1. 鲁棒全身 dexterous loco-manipulation 策略  
2. 易用可及的接口（如 VR）以规模化采数

## 三、方法详解

### 3.1 问题形式化

目标条件 RL / MDP：状态含本体感知 \(s^p_t\) 与目标 \(s^g_t\)（来自遥操作或自主智能体）。动作为关节目标角，PD 驱动。用 PPO 最大化回报。任务是**实时运动模仿**：\(\pi_{\text{OmniH2O}}\) 跟踪参考运动学 \(q_t=(\theta_t,p_t)\)。

### 3.2 人体运动重定向与数据偏置

从 AMASS 等大数据集重定向到人形（类似 H2O）。关键改进：为解决「小碎步调整、难以站稳」问题，对每条序列生成 **standing / squatting 下肢固定** 的稳定变体，偏置训练分布，强化站立与蹲姿下的上肢操作稳定性。

### 3.3 奖励与域随机

除模仿奖励外，正则奖励需**课程化**使用。关键奖励：**每步最大脚高度（max feet height）**——配合课程，帮助策略在「该站稳」与「该大步走」之间切换，避免为保平衡而跺脚。

### 3.4 教师-学生蒸馏（核心）

| | 教师 \(\pi_{\text{privileged}}\) | 学生 \(\pi_{\text{OmniH2O}}\) |
|:--|:--|:-|
| 训练 | PPO + 特权状态 | DAgger 监督蒸馏 |
| 目标 | 完整刚体位姿/速度差等 | 稀疏关键：头+双手位置（及差分） |
| 本体 | 完整仿真量 | 易真机获取量 + **25 步历史** |
| 线速度 | 可用 | **不依赖全局线速度**（历史隐式推断） |

学生本体示例：关节角/速、根角速度、重力投影、历史动作等，拼成长历史（约 1665 维）。遥操作时常用 VR 三点位置；其他接口也统一到三点以兼容商业设备，可扩展更多关键点。

**灵巧手**：VR 估计手姿 → IK → 现成底层手控制器；也可换成手套/RGB。

蒸馏：学生在仿真 rollout → 构造特权状态查教师动作 → L2 损失更新学生。消融表明：长历史用纯 RL 难训（成功率大跌），DAgger 才能吃下历史；25 步历史 + MLP 综合最优。

## 四、通用真机接口与自主

同一 \(\pi_{\text{OmniH2O}}\) 支撑：

1. **遥操作**：VR / RGB → 全身 + 灵巧手 loco-manipulation  
2. **语言**：文本 → MDM 生成运动目标 → 策略跟踪（挥手、T-pose、拥抱、前进、转向等）  
3. **鲁棒性**：拳踢扰动、草地/坡道/碎石等户外地形仍可稳  
4. **GPT-4o**：头戴相机图像 + 运动原语选择提示词 → 自主出拳、按姿态打招呼等（因延迟选原语而非直接出连续目标）  
5. **LfD**：高层策略 \(\pi_{\text{LfD}}\) 从图像预测稀疏运动目标序列（含手），再由 \(\pi_{\text{OmniH2O}}\) 做底层全身执行——**复用电机技能，大幅减少演示量**

### OmniH2O-6 数据集

六任务：Catch-Release、Squat、Rock-Paper-Scissors、Hammer-Catch、Boxing、Basket-Pick-Place。  
内容：头戴 RGBD、相对根的头/手运动目标、关节电机目标，约 30 Hz；合计约 **40 分钟**真机演示。

LfD 消融要点：

- 数据量↑ → MSE↓、成功率↑（约 25%→100% 数据：成功率约 4/10 → 8/10）  
- **动作序列预测**重要  
- Diffusion Policy（DDPM/DDIM）明显优于普通 BC-ResNet

## 五、实验结果摘要

### 仿真运动跟踪（增强 AMASS，约 14k 序列）

| 方法 | Succ↑ | 备注 |
|:-----|:------|:-----|
| Privileged 教师 | ~94.8% | 非直接部署 |
| H2O | ~87.5% | 基线 |
| OmniH2O 学生 | ~94.1% | 接近教师 |

三点跟踪相对 8/22 点略损精度，但可部署性大幅提升；显式加线速度在仿真无显著增益。

### 真机（20 条站立序列）

相对 H2O：全局/相对 MPJPE、加速度/速度误差全面更好（如 Eg-mpjpe 约 47.9 vs 87.3）。  
VIO 估速策略易摔；无显式线速度 + 历史的方案最稳。0 步历史真机误差明显变差，印证历史必要。

硬件：Unitree H1 + Damiao 电机 + Inspire 手；Vision Pro；ZED；双 Orin NX 或笔记本 ROS，策略约 50 Hz，系统延迟约 20 ms 量级。

## 六、局限与展望

- 仍需根里程计把接口姿态变换到机器人系；VIO 噪声/跳变会污染目标。  
- 极端扰动或目标不连续时缺乏安全保证。  
- 仅靠稀疏上肢目标爬楼梯等仍难。  
- 未来：更多传感（LiDAR、腕相机、触觉）与更好 LfD 算法。

---

## 七、学习要点（结合本阶段）

- **通用中间表示**：运动学姿态把 VR / RGB / 语言 / VLM / IL 接到同一底层。  
- **延迟与反馈**：稀疏输入靠策略「补全身」；无 MoCap 线速度靠历史，是野外部署关键。  
- **数据飞轮**：遥操作 → OmniH2O-6 → Diffusion Policy 等 → 自主；底层技能复用降低演示需求。  
- 对比 HOMIE：OmniH2O 是**端到端全身跟踪**，接口更「软」；HOMIE 是**解耦座舱 + 关节匹配**，上肢精度/成本路径不同。  
- 对比 TWIST：OmniH2O 用 VR 稀疏点；TWIST 用全身 MoCap，协同全身技能（蹲抬箱、踢球）更完整，但设备更重。

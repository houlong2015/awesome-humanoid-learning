# HOMIE: Humanoid Loco-Manipulation with Isomorphic Exoskeleton Cockpit

> 论文：Qingwei Ben, Feiyu Jia, Jia Zeng, Junting Dong, Dahua Lin, Jiangmiao Pang  
> 机构：上海人工智能实验室 / 香港中文大学多媒体实验室  
> arXiv：[2502.13013](https://arxiv.org/abs/2502.13013)（v2, 2025-04）  
> 标签：`Teleoperation` · 同构外骨骼座舱 · 半自主全身遥操作

---

## 一、总结

HOMIE 提出一套**半自主人形机器人遥操作座舱**：用 RL 下肢策略（踏板发命令）+ **同构外骨骼臂**（关节直接匹配）+ **霍尔传感手套**（灵巧手），让单人高效完成行走、蹲下与上肢操作一体的 loco-manipulation，并形成数据采集飞轮。

核心卖点：

1. **效率高**：关节匹配，无需 IK / 视觉估姿；桌面任务完成时间约为 OpenTelevision（VR）的一半。  
2. **工作空间大**：策略支持任意上肢姿态下的行走与**任意高度蹲起**，可够高够低。  
3. **成本低**：整套硬件约 **$500**，开源。  
4. **数据可用**：采集演示后可用模仿学习（Seer）训出自主策略（蹲捡约 73%、拾放约 80% 成功率）。

---

## 二、问题背景

人形 loco-manipulation 需要：**全身协调** + **接触丰富的精细操作**。现有路线分裂：

| 方向 | 优点 | 不足 |
|:-----|:-----|:-----|
| RL 运动策略 | 环境适应强 | 缺少实时精确遥操作接口 |
| 多数遥操作系统 | 上肢可控 | 往往忽略下肢，工作空间受限 |

HOMIE 的目标是：RL 训练时显式兼容上肢遥操作接口，同时硬件侧把运动与上肢控制统一进一个「座舱」。

---

## 三、系统总览

通信：座舱 ↔ 机器人经 **Wi-Fi**；操作者看机器人 **FPV**。

| 模块 | 输入 | 作用 |
|:-----|:-----|:-----|
| 踏板 | \(C_t=[v_x,\omega_{yaw},h]\) | 线速度、转向、躯干高度 |
| \(\pi_{loco}\) | 本体感知 + 命令历史 | 控制**下肢**（PD 跟踪策略动作） |
| 同构外骨骼 | 关节角 \(q_{upper}\) | **直接设定**机器人上肢关节 |
| 传感手套 | 最多 15 DoF 手指角 | 映射各类灵巧手 |

数据飞轮：遥操作采集演示 → 训练自主策略 \(\pi_{auto}\) → 接管操作者给出 \(C_t\) 与 \(q_{upper}\)。

## 四、方法详解

### 4.1 RL 全身下肢策略 \(\pi_{loco}\)

在 Isaac Gym 中用 PPO 训练，可 zero-shot sim-to-real。以 Unitree G1 为例，也验证了 Fourier GR-1。

**观测**：\(O_t=[C_t,\omega_t,g_t,q_t,\dot q_t,a_{t-1}]\)，拼接 \(O_{t-5:t}\) 作为输入。  
**动作**：下肢关节目标，经 PD 产生力矩：

\[
\tau_{t,i}=K_{p,i}(a_{t,i}-q_{0,t,i})-K_{d,i}\dot q_{t,i}
\]

**三大训练技巧**：

#### （1）上肢姿态课程（Upper-body Pose Curriculum）

训练中上肢关节角按课程逐渐加大扰动范围，用比率 \(\rho_a\)（从 0→1）控制采样；并用平滑概率分布保证全程都能采到 \([0,1]\) 区间内的姿态，每 1 秒重采样目标姿态并做均匀插值，模拟座舱连续挥动上肢。消融显示：无课程或简单均匀裁剪课程，速度/高度跟踪都更差。

#### （2）高度跟踪奖励（Height-tracking）

引入膝相关奖励 \(r_{knee}\)，在身高低于/高于目标时分别鼓励屈膝/伸膝，从而学会蹲到指定高度 \(h_t\)。训练时约 1/3 环境学蹲、2/3 学站走，并在环境间切换，保证蹲↔走平滑过渡。仅放大高度 reward 会损害其他目标；\(r_{knee}\) 收敛更快、更稳。

#### （3）对称性利用（Symmetry）

对 transition 做左右镜像增强，并加 actor/critic 对称损失 \(L^{actor}_{sym}\)、\(L^{critic}_{sym}\)，提高数据效率、减轻左右不对称。

**相对先前 WBC**：不依赖 MoCap motion prior，管线更轻。

### 4.2 硬件：同构外骨骼座舱

#### 同构外骨骼臂（约 $430）

- 每侧 **7 DoF**（肩 3 + 肘 1 + 腕 3），对齐 G1 / GR-1 上肢形态。  
- Dynamixel XL330 伺服读角（约 0.09°），可掉电保持绝对位置，免反复标定。  
- **关节匹配**：\(q_t=\pm k_t(p_t+n_t\pi/2)+\tau_t\)，绕过 IK 与视觉估姿 → 更快、更准。

#### 运动传感手套（约 $30/只）

- 基于霍尔传感器 + 钕磁铁，每指多组传感，合计最多 **15 DoF**。  
- 可拆卸，复用到不同同构臂；可在仿真中驱动 LEAP / Shadow / Inspire / Schunk 等多种手。

#### 踏板（约 $20）

- 三个小踏板分别映射线速度、偏航、高度；模式键切换前后/左右。  
- 解放双手，无需与机器人同步行走。

**频率对比（上肢输出到机器人）**：HOMIE 臂/手约 **263 / 293 Hz**，无需 GPU/SoC；对比 Telekinesis、AnyTeleop、OpenTeleVision 等视觉/VR 方案明显更高。

## 五、实验与结果

### 5.1 策略消融与跨机器人

指标：线/角速度误差、高度误差、对称损失、存活时间。课程、\(r_{knee}\)、对称三项均有效。G1 与 GR-1（身高、重量、手重比差异大）均可在相同管线下收敛到可用策略。

### 5.2 真机遥操作任务（图 1）

包括：蹲捡胶带放到高架、双机递苹果、搬箱换架、推坐椅上的人（约 60 kg）、开烤箱门、双手协作递放番茄、地面捡瓶、插花入箱、剧烈上肢扰动下平衡等。单操作员 + Wi-Fi（约 16 ms 延迟，包长约 128 bytes）。

### 5.3 效率与用户研究

- 四项桌面任务（Pick&Place、扫码、Hand Over、开烤箱）相对 OpenTelevision，完成时间约 **一半**。  
- 5 名新手做 Hand Over：短教程后数次试验即可接近专家速度；体型差异可用绑带适配。

### 5.4 模仿学习闭环

- 任务：Squat Pick、Pick & Place；各 50 episode，10 Hz 采 RGB + \(q_t\) + \(q_{upper}\) + \(C_t\)。  
- 模型：Seer（自回归 Transformer + MAE ViT），真机成功率约 **73.3% / 80.0%**。

仿真侧可迁移到 GRUtopia / Isaac Sim 场景做虚拟遥操作。

## 六、贡献与局限

**贡献**：

1. RL 下肢 + 同构外骨骼 + 手套的统一座舱，单人全身控制。  
2. 首批无需 motion prior 的、可遥操作兼容的蹲走 loco-manipulation 实现之一。  
3. 低成本、高精度、高频硬件；任务更快完成。

**局限**：

- 多样地形穿越仍不足。  
- 手套拇指设计与人手解剖不完全对齐，部分灵巧手操作不够顺滑。  
- **无力反馈**；外骨骼**未覆盖腰部**遥操作。

---

## 七、学习要点（结合本阶段）

- 数据链路：**人 → 踏板/外骨骼/手套 → 关节匹配 + RL 下肢 → 机器人 → 录制 → IL**。  
- 同构关节匹配直接规避 IK/估姿误差，是「人-机具身差异」的一种硬件解法。  
- 蹲高控制扩展工作空间；对称与课程是让下肢在上肢大幅扰动下仍稳的关键。  
- 与 OmniH2O / TWIST 对比：HOMIE **上/下解耦**（踏板 + 上肢关节），换取座舱成本与精度；全身协同动作（如踢球开门）不如全身模仿系统自然。
